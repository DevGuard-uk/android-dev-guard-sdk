```kotlin
}
```

**JitPack** remains a fallback; Maven Central is the official listing.

## Step 1 — Sonatype Central account

1. Open **[https://central.sonatype.com](https://central.sonatype.com)**.
2. **Sign in** → **Sign in with GitHub**.

---

## Step 2 — Register namespace `uk.devguard`

1. **Publish** → **Namespace** (or [central.sonatype.com/publishing](https://central.sonatype.com/publishing)).
2. Enter **`uk.devguard`**.
3. Verification: **Domain** → **`devguard.uk`**.
4. Add Sonatype’s **TXT** record in Cloudflare (DNS → `@` for `devguard.uk`).

Wait until namespace shows **Verified** before publishing.

## Step 3 — GPG signing key

Maven Central requires **signed** artifacts (`.asc` for each file). See [requirements](https://central.sonatype.org/publish/requirements/).

### Generate (interactive)

```bash
brew install gnupg   # if needed
gpg --full-generate-key
```

Suggested answers:

| Prompt | Value |
|--------|--------|
| Kind | `(1) RSA and RSA` |
| Size | `4096` |
| Expiry | `0` (no expiry) or `2y` |
| Name | `DevGuard UK` |
| Email | `app@devguard.uk` |
| Passphrase | Save in password manager |

### Or generate (batch — automation)

```bash
cat > /tmp/gpg-batch <<'EOF'
%echo Generating DevGuard UK signing key
Key-Type: RSA
Key-Length: 4096
Subkey-Type: RSA
Subkey-Length: 4096
Name-Real: DevGuard UK
Name-Email: app@devguard.uk
Expire-Date: 0
Passphrase: YOUR_PASSPHRASE_HERE
%commit
%echo done
EOF
gpg --batch --generate-key /tmp/gpg-batch
rm /tmp/gpg-batch
```

### List key + upload public key

```bash
gpg --list-secret-keys --keyid-format=long
gpg --keyserver keys.openpgp.org --send-keys YOUR_FULL_KEY_ID
# optional: gpg --keyserver keyserver.ubuntu.com --send-keys YOUR_FULL_KEY_ID
```

**No GPG UI in the Central Portal.** The old OSSRH “Account → GPG Keys” page is gone. Sonatype validates signatures against your **public key on a keyserver** during deployment. Propagation can take up to an hour.

Verify upload:

```bash
curl -s "https://keys.openpgp.org/vks/v1/by-fingerprint/YOUR_FINGERPRINT_NO_SPACES" | head -3
```

## Step 4 — Sonatype user token

1. [central.sonatype.com](https://central.sonatype.com) → profile menu (top right).

3. Copy **username** and **password** (token secret) — shown once.

## Step 5 — Local Gradle credentials (never commit)

### File layout

| File | Committed? | Contents |
|------|------------|----------|
| `signing-keys.properties` | Yes | Org GPG key ID, fingerprint, namespace (public) |
| `gradle-publish.local.properties` | **No** (gitignored) | Sonatype token + GPG passphrase |
| `~/.gradle/gradle.properties` | Local machine | Generated by setup script |

### Automated setup

```bash
cd deployment/native-android-dev-guard
cp gradle-publish.local.properties.example gradle-publish.local.properties
# Edit: mavenCentralUsername, mavenCentralPassword, signingGpgPassphrase
chmod +x scripts/setup-gradle-publish-props.sh
./scripts/setup-gradle-publish-props.sh
```

Project flag (already set in repo):

```properties
# deployment/native-android-dev-guard/gradle.properties
mavenCentralPublishing=true
```

## Step 6 — Publish

```bash
cd deployment/native-android-dev-guard

# Dry run — build only
./gradlew :sdk:assembleRelease :core:assembleRelease :crash-reporter:assembleRelease

# Upload + sign all modules
./gradlew publishAllToMavenCentral
```

Published artifacts:

| Artifact | Coordinates |
|----------|-------------|
| Main SDK | `uk.devguard:android-sdk:1.0.1` |
| JNI core | `uk.devguard:android-core:1.0.1` |
| Crash module | `uk.devguard:android-crash-reporter:1.0.1` |

### Portal after upload

1. [central.sonatype.com/publishing](https://central.sonatype.com/publishing) → **Deployments**.
2. Wait for **VALIDATED** (or **PUBLISHED** if auto-publish).
3. If **VALIDATED**, click **Publish** in the UI.
4. Verify (~15–30 min): [artifact/uk.devguard/android-sdk](https://central.sonatype.com/artifact/uk.devguard/android-sdk).

## Technical notes

- **Publish URL:** Gradle uses Sonatype’s OSSRH compatibility endpoint:  
  `https://ossrh-staging-api.central.sonatype.com/service/local/staging/deploy/maven2/`
- **Portal transfer (required):** `maven-publish` only stages files. `publishAllToMavenCentral` now calls `transferMavenCentralStagingToPortal`, which POSTs to Sonatype’s manual API so the deployment appears under **Deployments**. Without this step the portal stays empty even when Gradle reports success. Docs: [publish-portal-ossrh-staging-api](https://central.sonatype.org/publish/publish-portal-ossrh-staging-api/).
- **Signing:** Root `build.gradle.kts` calls `useInMemoryPgpKeys()` when `signingInMemory*` props are set.
- **Imports:** Maven `groupId` is `uk.devguard`; Kotlin/Java packages remain `io.devguard.*`.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Deployments tab empty | Run `./gradlew transferMavenCentralStagingToPortal` (auto-runs after `publishAllToMavenCentral`) |
| DNS verification fails | TXT on Cloudflare `@` for `devguard.uk`; wait 30 min |
| `401 Unauthorized` | Regenerate token; re-run `setup-gradle-publish-props.sh` |
| `404` on publish | Wrong repository URL — see Technical notes above |
| Signing failed | Re-run setup script; confirm key on keys.openpgp.org |
| Deployment stuck | Wait 1–2 h; contact central-support@sonatype.com |
